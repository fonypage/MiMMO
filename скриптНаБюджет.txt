ЗАДАНИЕ 1: Диаграмма переходов состояний

Ниже представлена диаграмма в текстовом виде (можно перерисовать в Excel или Word):

```

                      │              │
                ┌────▶│    OFF       │◄────┐
                │     │ (Выключена)  │     │

                │            │             │

                │    Вода+Зерна│             │

                └─────│    READY     │─────┘
                      │   (Готова)   │
                      └──────────────┘
                              │
                              │ START
                              ▼
                      ┌──────────────┐
                      │              │
                      │   BREWING    │

                              │
```

Описание переходов стрелками:

1. OFF → READY: POWER (есть вода и зерна)
2. READY → BREWING: START
3. BREWING → READY: Прошло 2 минуты
4. READY → ERROR: Закончилась вода ИЛИ закончились зерна
5. ERROR → READY: Долита вода И добавлены зерна
6. OFF ← → ERROR: POWER (3 сек)
7. READY ← → OFF: POWER (3 сек)
8. BREWING ← → OFF: POWER (3 сек)
9. ERROR ← → OFF: POWER (3 сек)

ЗАДАНИЕ 3: 5 наиболее критичных тестовых сценариев

№ Сценарий Почему критично
1 OFF → READY → BREWING → READY (нормальный цикл) Проверяет основной бизнес-процесс — приготовление кофе
2 READY → ERROR → READY (восстановление после ошибки) Проверяет, что после устранения проблемы кофемашина возвращается в работу
3 BREWING → ERROR (авария во время приготовления) Проверяет безопасность — что происходит при внезапной ошибке
4 Любое состояние → OFF (долгое нажатие POWER) Проверяет аварийное выключение из любого состояния
5 OFF → ERROR (включение без воды/зерен) Проверяет корректную обработку ошибки при старте

---

ЗАДАНИЕ 4: 2 "Зловредных" теста (mischievous tests)

Тест №1: Попытка приготовить кофе в состоянии ERROR

· Действие: Кофемашина в состоянии ERROR (нет воды). Нажимаем кнопку START.
· Ожидаемый результат: Кофемашина не должна переходить в BREWING, должна остаться в ERROR или показать сообщение, что действие недоступно.
· Почему зловредный: Пользователь может попытаться запустить приготовление, не устранив ошибку.

Тест №2: Многократное быстрое переключение POWER

· Действие: Быстро нажимать и отпускать кнопку POWER (не 3 секунды) в разных состояниях.
· Ожидаемый результат: Короткие нажатия не должны выключать кофемашину (только долгие). Кофемашина должна игнорировать ложные срабатывания.
· Почему зловредный: Проверяет защиту от дребезга контактов или случайных нажатий.
